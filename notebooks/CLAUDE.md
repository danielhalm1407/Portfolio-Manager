# notebooks/ — Claude Instructions

## Structure
- `exploration/` — one-off analysis, ideation, messy experimentation
- `reports/` — polished output notebooks for review and presentation

## Conventions
- Exploration notebooks are disposable — don't over-engineer them
- Report notebooks should be reproducible: restart kernel and run all before committing
- If analysis in `exploration/` proves useful, extract the logic into `src/portutils/`
